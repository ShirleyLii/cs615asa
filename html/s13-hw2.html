CS615A -- Aspects of System Administration - Spring 2013 - HW#2

HW#2: Package Management on different OS

Summary:

The objective of this assignment is for you to install a few software packages on a few different operating systems or operating system flavors. You will learn to use the standard package management tools of each as well how to build software manually as well as how to create a package for a piece of software to install on another system.

At the same time, you will practice using the EC2 command-line tools to create and administer the different OS instances. Writing the documentation to perform these tasks will again be part of your assignment.

This is a longer assignment consisting of multiple parts; it is worth 75 points.

OS Instances

The exercises below will be done once for each of the following operating systems/AMIs:

  • "Getting Started on Fedora Core 8" (AMI Id: ami-b232d0db)
  • Ubuntu 13.04 (AMI Id: ami-1fb63576, or see this page)
  • OmniOS stable r151004 (AMI Id: ami-4e2c9727)
  • a FreeBSD instance (AMI Id: ami-9d38baf4, or see this page)
  • a NetBSD instance (AMI Id: ami-5d0f8034, or see this page)

Creating the instance

Using the EC2 command-line tools, create an instance of each of the above listed AMIs. (You can do this in parallel, or sequentially, as you prefer.) Familiarize yourself with all available command-line tools, most notably, of course, the following commands:

  • ec2-describe-images
  • ec2-add-group
  • ec2-describe-group
  • ec2-authorize
  • ec2-run-instances
  • ec2-describe-instances
  • ec2-get-console-output

Install software using the native package managemer

25 points

On each instance, use the OS's native package manager (finding out whichever that is and how to use it is part of the assignment) to install the following packages (if not already installed):

  • gcc
  • perl
  • python
  • gpg (sometimes known as gnupg)
  • screen
  • sudo
  • sipcalc

Though not part of this assignment, you probably should familiarize yourself with these tools.

Install software "by hand"

25 points

Choose any one of the Linux systems listed above as well as one of the non-Linux system. Then, install Graphite on each without using the package manager. That is, download the software and follow the instructions provided by the software to install it.

Create an RPM for "Statsd"

25 points

On an RPM based Linux system (such as the Fedora instance noted above), create an RPM for the Statsd software package. Verify that the resulting package can be installed on a new instance.

Your submission for this part of the homework will include the .spec file you created.
